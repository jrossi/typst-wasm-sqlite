# CLAUDE.md

## Project Overview

Typst WASM plugin that embeds SQLite for compile-time database queries. Zig compiles SQLite + a custom read-only VFS into a ~150KB WASM module. Typst reads database files as bytes, passes them to the plugin, and gets JSON back.

## Tech Stack

- **Zig 0.15.2** — builds WASM target (`wasm32-freestanding`) and native test/gen executables
- **Typst 0.14.2** — document compiler, uses WASM minimal plugin protocol
- **SQLite 3.47.2** — vendored amalgamation (`sqlite3.c`), compiled as C dependency
- **mise** — tool version manager and task runner (`mise.toml`)

## Key Architecture

- `src/main.zig` — WASM entry points (`query`, `tables`, `schema`, `hello`), custom read-only VFS, memory allocator
- `src/json.zig` — Fixed-buffer JSON writer (no heap allocation, overflow-safe)
- `sqlite.typ` — Typst-side API wrapper (reads db bytes, calls plugin, parses JSON)
- `libc-stubs/` — Minimal libc implementation for WASM freestanding (no OS)
- `build.zig` — Builds WASM plugin + native test runner + database generators

## Build Commands

```bash
zig build -Doptimize=ReleaseSmall   # Build WASM plugin (release)
zig build                            # Build WASM plugin (debug)
zig build test                       # Run unit tests (~35 JSON serializer tests)
zig build gen-testdb                 # Generate test/test.sqlite
zig build gen-demodb                 # Generate demo.sqlite
mise run test                        # Full suite: unit + integration
mise run demo                        # Build + compile demo report
```

## Testing

- **Unit tests**: `tests/test_json.zig` tests the JSON serializer (imported as module via build.zig)
- **Integration tests**: `test/test_plugin.typ` and `test/test_api.typ` use Typst `assert.eq()` — compile failure = test failure
- **Test runner**: `test.typ` includes both test files
- **Test databases**: generated by native Zig executables that link sqlite3.c directly
- **Git hooks**: pre-commit runs unit tests, pre-push runs full suite (configured in mise.toml)

## Important Patterns

- Plugin functions return 0 for success, 1 for error. Return code 1 causes a Typst **compile error** — error JSON is never parsed by Typst.
- `(db.query)(sql)` parenthesization is required in Typst for calling dict methods.
- JsonWriter uses a fixed buffer with overflow detection — `getResult()` returns `?[]const u8` (null on overflow).
- `isValidTableName()` guards against SQL injection in `schema()` — only allows `[a-zA-Z_][a-zA-Z0-9_]*`.
- SQLite is configured with many OMIT flags to minimize WASM size (no WAL, no UTF16, no shared cache, etc.).

## File Conventions

- `src/` — Zig source for the WASM plugin
- `tests/` — Zig test files and database generators (native executables)
- `test/` — Typst integration test files
- Root `.typ` files — user-facing documents (example, demo, test runner, API wrapper)
